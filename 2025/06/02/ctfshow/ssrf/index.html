SSRF的运用原理:

1.原理:

利用存在漏洞的服务器，向内部或外部资源发送伪造的请求，从而达到以下目的：

  • 探测内网结构（端口扫描、服务发现）
  • 访问内网API（如云服务元数据接口）
  • 进行身份伪造（访问携带身份信息的资源）
  • 造成DoS、反弹请求等攻击

2.常用检测方法:

检测类型方法描述检测是否成功的方式绕过思路
🌐 常见访问内网地址127.0.0.1、localhost、169.254.169.254、内网IP段等返回内容、响应时间差使用整数、16进制、[::]等变体绕过
🌐 常见DNSlog平台检测传入外部 DNS 地址（如 xx.ceye.io）DNS平台记录到解析请求域名跳转、伪协议等规避安全策略
🌐 常见HTTP外带检测构造 URL 发起外部 HTTP 请求服务器访问日志、Burp Collaborator、tcpdump基于 SSRF 的跳转链
🌐 常见文件协议file:///etc/passwd 等如果读取内容返回，证明存在 SSRFfile协议黑名单绕过、伪协议组合
🌐 常见gopher协议检测构造 Redis 注入 / SSRF-RCE返回内容为空，但配合后端验证成功被多数防火墙限制，注意协议混淆
🌐 常见请求延迟检测访问 http://10.0.0.1:81 等，观察延迟请求阻塞 / 超时 / 响应慢用于端口扫描时做判断
⚠️ 不常见请求头注入检测伪造 Host, X-Forwarded-Host, X-Real-IP 等后端错误信息中暴露 Host 信息配合 Host header 漏洞打 SSRF
⚠️ 不常见Open Redirect 联动检测先跳转再 SSRF可被 SSRF 利用的 OpenRedirect常规过滤难以识别 URL 跳转链
⚠️ 不常见SVG 渲染触发 HTTP 请求提交嵌入外链的 SVG，观察目标服务器是否发起请求需要精心构造图像并外带链接绕过非URL型白名单
⚠️ 不常见PDF生成功能 SSRF提交带图像URL的 HTML，让服务器渲染 PDF 触发请求外部服务器或DNS记录PDF服务通常可执行外部请求
⚠️ 不常见OAuth/SSO 重定向利用借助SSO第三方回调URL发起 SSRF需要结合OAuth实现细节重定向 + SSRF合击链
⚠️ 不常见SMTP SSRFsmtp://127.0.0.1:25 向邮件服务注入请求邮件被发送、行为侧信道多协议SSRF + 内网钓鱼链

3.linxu常见敏感目录:

路径含义
/etc/passwd用户信息（无密码）
/etc/shadow加密密码（需root权限）
/root/.ssh/id_rsaSSH私钥
/var/www/html/config.phpWeb站点配置
/proc/self/environ当前进程环境变量（可能泄漏Token）
/proc/version内核版本信息

1.verb（格式化动词）

1.常用 verb 格式化动词表

动词含义示例输出（假设变量值为 x = 42）
%v默认格式输出42
%+v输出结构体时，包含字段名{Name:Tom Age:18}
%#v输出 Go 语法表示的值main.User{Name:"Tom", Age:18}
%T显示变量类型int
%%输出 % 本身%

2.数值类型格式化 verb

动词说明示例 (x = 42)
%d十进制整数42
%b二进制整数101010
%o八进制52
%x十六进制（小写）2a
%X十六进制（大写）2A
%cUnicode 字符*（如果 42）

动词说明示例 (s = "hello")

3.常量:

标识符类型是否为常量用途说明示例
truebool✅ 是常量表示布尔真值const ok = true
falsebool✅ 是常量表示布尔假值var done = false
iotaint⚠️ 特殊标识符用于 常量声明组中自动生成递增数字（仅在 const 中可用）const (A = iota; B; C)
nil预定义空值⚠️ 不是 const 常量表示指针、切片、map、chan、接口、函数的零值（未初始化）var p *int = nil

声明方式:

声明方式示例说明是否支持类型推导
1. 显式声明（带类型）var a int默认值为0（int）❌ 不支持
2. 显式声明 + 初始化var a int = 10有初始值✅ 支持（可省略类型）
3. 类型推导声明var a = 10类型自动为 int✅
4. 短变量声明a := 10只能在函数内部使用✅
5. 多变量声明var a, b int = 1, 2可多个变量同时声明✅
6. 匿名变量_ = someFunc()用于忽略返回值❌
7. 常量声明const Pi = 3.14值不可修改✅（编译期推导）

11.面向对象

Go 语言不是典型的面向对象语言，但它通过 结构体 + 接口 实现了封装、继承和多态的能力。

特性Go实现方式示例特点
封装字段/方法首字母大写表示公开，小写表示私有Name string vs name string访问控制基于大小写
继承通过组合（Embedding）结构体实现“继承”type Student struct { Human }没有 extends，但有字段/方法提升
多态通过接口 interface 实现type Speaker interface { Speak() }接口隐式实现，无需关键字

12.结构体标签:

在 Go 语言中，结构体标签（Tag） 是给结构体字段添加额外元信息的一种方式，最常见的应用场景是用于 JSON 编解码、数据库ORM映射、验证器等。

项目示例说明
语法格式字段名 类型标签名:”值”标签必须用反引号 `
JSON 标签json:"name"用于控制字段序列化时的名称
多标签json:"name" db:"user_name"多个标签中间用空格分隔
忽略字段json:"-"被 - 忽略，不参与序列化
结构体字段首字母必须大写Name string小写字段无法导出，标签也无效

标签的用途

用途标签示例第三方库
JSONjson:"name"encoding/json
数据库ORMgorm:"column:user_name"gorm
表单验证validate:"required"go-playground/validator
配置解析yaml:"port"gopkg.in/yaml.v2
结构体比较cmp:"ignore"github.com/google/go-cmp

13.go协程(goroutine):

1.基础用法:

用法场景语法示例
启动协程go 函数名()go doWork()
匿名函数协程go func(){...}()go func(){ fmt.Println("Hi") }()
协程同步sync.WaitGroup用于等待所有协程完成
协程通信chan 通道ch := make(chan int)，用于发送/接收数据
协程超时select + time.After处理阻塞操作的超时控制

2.注意事项:

注意点原因
不要在 goroutine 中访问共享变量（需加锁或使用 channel）避免数据竞争（data race）
主程序要等待协程完成，否则程序会提前退出使用 sync.WaitGroup
不要忘记 wg.Done()，否则会死锁wg.Wait() 永远不会结束
大量协程可用 GOMAXPROCS 控制核心数runtime.GOMAXPROCS(n)

14管道(channel):

1.基本使用:

概念说明示例
创建通道用 make(chan 类型) 创建ch := make(chan int)
发送数据使用 <- 向通道发送值ch <- 10
接收数据从通道中读取值val := <-ch
无缓冲通道发送时必须等待接收方ch := make(chan int)
有缓冲通道可存储一定数量数据ch := make(chan int, 3)
关闭通道表示不会再发送数据close(ch)
for range 读取读取所有通道数据直到关闭for v := range ch {...}

2.与select使用:

使用模式示例代码片段说明
接收多个通道数据select { case v := <-ch1: ... case v := <-ch2: ... }等待多个输入，哪个先来就处理哪个
超时控制select { case <-ch: ... case <-time.After(t): ... }避免 goroutine 永远阻塞
退出信号监听select { case <-data: ... case <-done: return }通过 done 通道优雅退出
非阻塞 send/recvselect { case ch <- val: ... default: ... }如果通道阻塞就执行 default

1.linxu系统文件加密:

技术全称工作原理特点安全/攻击要点
AFSAndrew File System分布式文件系统，最早由 CMU 开发，支持跨网络的统一命名空间。AFS 本身并非主要加密实现，但结合 Kerberos 提供认证、访问控制。优势在于分布式共享、集中认证；加密主要依赖外部机制。攻击者可尝试 Kerberos 票据窃取/重放（TGT/TGS）；配置不当易出现明文传输。
TCFSTransparent Cryptographic File SystemLinux 内核模块，文件系统层实现透明加密，用户无需手工调用加解密。密钥绑定到用户身份。使用 NFS 的扩展，文件透明加解密，用户体验较好。早期算法较弱（如 DES）；若密钥管理不当，root 权限可直接绕过。
CFSCryptographic File System基于用户空间（借助 NFS loopback）的文件加密系统。需要用户挂载时输入密码。简单、易用，文件透明加解密。因运行在用户空间，性能较差；密码暴力破解风险高。

2.windows系统文件加密:

技术全称工作原理特点安全/攻击要点
EFSEncrypting File SystemWindows 内置，NTFS 的扩展功能。使用对称加密（AES、以前是 DESX），文件加密密钥 (FEK) 再用用户公钥（RSA）加密存储。与 NTFS 无缝集成；用户/管理员可透明访问加密文件。攻击者若拿到用户私钥（存放在证书存储中，常在 %APPDATA%\Microsoft\Crypto），即可解密；可通过 DPAPI/密钥导出工具（如 mimikatz）窃取。
